# Basics

A `Schematic` is a named collection of blocks, block entities, entities, and
metadata held in one or more regions. Create one from scratch or open an
existing file, place blocks using ordinary Minecraft block-state strings,
inspect the result by coordinate, and save it again:

```python
from nucleation import Schematic

build = Schematic.create("crafting_nook")

for x in range(5):
    for z in range(5):
        build.set_block(x, 0, z, "minecraft:spruce_planks")

for y in (1, 2, 3):
    for x in range(5):
        if x == 2 and y == 2:
            block = "minecraft:light_blue_stained_glass"
        elif x in (0, 4):
            block = "minecraft:stripped_spruce_log[axis=y]"
        else:
            block = "minecraft:oak_planks"
        build.set_block(x, y, 0, block)

    for z in range(1, 5):
        if z == 2 and y == 2:
            block = "minecraft:light_blue_stained_glass"
        elif z == 4:
            block = "minecraft:stripped_spruce_log[axis=y]"
        else:
            block = "minecraft:oak_planks"
        build.set_block(0, y, z, block)

build.set_block(1, 1, 1, "minecraft:crafting_table")
build.set_block(3, 1, 1, "minecraft:chest[facing=south]")
build.set_block(4, 2, 1, "minecraft:wall_torch[facing=south]")
build.set_block(1, 2, 4, "minecraft:wall_torch[facing=east]")

build.save_to_file("crafting-nook.schem")

copy = Schematic.load_from_file("crafting-nook.schem")
block = copy.get_block(1, 1, 1)
block.name()                          # "minecraft:crafting_table"
```

## Coordinates and automatic growth

Coordinates are integer block centres. A newly created schematic grows as
needed when a placement falls outside its current region, so there is no size
allocation step before construction.

Negative coordinates work too:

```python
build.set_block(-1, 0, 0, "minecraft:oak_planks")
build.get_block(-1, 0, 0).name()      # "minecraft:oak_planks"
```

## Block states

Use the same block-state strings Minecraft tooling uses. Properties go in
brackets and round-trip with the schematic:

```python
build.set_block(
    0, 1, 0,
    "minecraft:lever[face=wall,facing=east,powered=false]",
)
```

`get_block` returns the complete `BlockState`. `get_block_name` and
`get_block_string` are shortcuts for callers that only need the identifier or
serialised state. The broader API also exposes palette access, regions, block
entities, and typed entities;
see [Regions, transforms, and stamping](regions-and-transforms.md) and
[Block entities, entities, and NBT](block-entities-nbt.md).
